# 今天吃啥不胖

一个网页可行性原型，用来验证「减脂/健身人群的吃饭决策器」这个产品切口。

## 当前版本

- 首屏就是可用的吃饭决策器
- 支持填写性别、年龄、身高、体重、目标和今日已吃
- 支持输入多个想吃选项，并输出推荐排序、适配分和具体吃法
- 支持 30 个高频吃饭场景模板
- 支持记录推荐是否有用、最后实际吃了什么
- 支持部署后通过 `/api/geo` 读取粗略城市/时区，做轻量个性化提示
- 支持收藏常吃组合
- 页面补充了产品定位、场景、补偿方案、MVP 路线

## 运行方式

直接用浏览器打开 `index.html` 即可。

如果要通过微信传文件给手机测试，优先发送 `fit-eat-standalone.html`。这是单文件版，样式和脚本都内嵌在一个 HTML 里，不依赖同目录下的 CSS/JS 文件。

本地启动网址：

```bash
npm start
```

然后打开 `http://localhost:5173/`。

## 公网部署

项目已经补了 `vercel.json` 和 `api/geo.js`，推荐部署到 Vercel。部署后，Vercel 会把访问者的粗略地理信息放进请求头，`/api/geo` 会返回城市、地区、时区等非精确信息。正式验证用户时建议使用稳定公网域名；临时隧道只适合短时间测试。

隐私边界：页面只使用粗略城市/时区做提示，不展示完整 IP，不做疾病或医疗判断。

## 后续 App 化

`logic.js` 是独立推荐逻辑，可以迁移到 React Native + Expo。第一版 App 可以复用当前的输入、排序、收藏和样例场景，再逐步加入高频食物库和 AI 文案层。
